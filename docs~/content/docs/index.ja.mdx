**AchReactive** は、スキル・モンスター・アイテム・マップ・クエストなど、あらゆるゲームコンテンツを
**トリガー → 条件 → 効果** のデータの組み合わせで作成するUnityフレームワークです。

## 核心となるアイデア

> ドメインは無限だが、動作（primitive）は有限である。

ドメインごとにシステムを新しく作ると、`도메인 × 동작` マトリクスを毎回埋める必要があります。
AchReactive は動作プールを**すべてのドメインが共有**するようにし、この掛け算を足し算に変えます。
2つ目のドメインからはほぼコストゼロです。

```
        damage  heal  add_modifier  spawn
Skill     ✓      ✓        ✓
Monster                    ✓          ✓     ← 같은 effect 재사용
Item               ✓       ✓
```

## 特徴

- **ドメイン非依存** — `DataBase<T>` 一つで Skill/Monster/Item/… をすべて処理
- **ソース非依存** — JSON・CSV・リモートなどどこからでもロード（`IDataLoader<T>` を実装するだけで新しいソースに対応）
- **動作の自動登録** — `[Effect]`/`[Condition]`/`[Trigger]` メソッドをリフレクションで収集、配線0行
- **UIToolkit オーサリングツール** — 型とデータをエディタで生成（JSONの手書き不要）
- **依存性最小** — Newtonsoft.Json 一つ

## ひと目で見る

```csharp
// 1) 動作プールの自動収集（基本の effect/condition/trigger を含む）
EffectRegistry.AutoRegister();
ConditionRegistry.AutoRegister();
TriggerRegistry.AutoRegister();

// 2) データの読み込み（ソースは自由）
var skills = new DataBase<SkillData>();
skills.Load(new JsonDataLoader<SkillData>(() => json));

// 3) 実行 — ドメインが異なっても同じ
engine.Run("on_use", skills.Get("fire_slash"), ctx);
```
