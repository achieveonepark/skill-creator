---
title: 버프 시스템
description: 스택·갱신 정책, 주기 효과, 스탯 수정자의 런타임 동작.
---

버프는 `BuffController` 가 중앙에서 관리합니다. 적용·스택·갱신·주기 효과·해제와 스탯 수정자 적용을 모두 담당합니다.

## 적용과 해제

```csharp
// 효과를 통한 적용 (스킬 effect: add_buff)
system.Buffs.AddBuff(target, "burn", source: caster, overrideDuration: 5f);

// 직접 제거
system.Buffs.RemoveBuff(target, "burn");
```

적용 시 `onApplyEffects`, 해제 시 `onRemoveEffects` 가 1회 실행됩니다. 해제 시점에는 적용했던 스탯 수정자가 정확히 원복됩니다.

## 스택 정책 (stackPolicy)

이미 같은 버프가 존재할 때의 동작입니다.

| 정책 | 동작 |
| --- | --- |
| `stack` | `maxStack` 까지 스택 증가. 스탯 수정자도 스택 배율로 재적용 |
| `ignore` | 새 적용 무시 (갱신 정책만 반영) |
| `replace` | 기존을 대체, 스택 1 유지 |

## 갱신 정책 (refreshPolicy)

재적용 시 남은 지속시간 처리입니다.

| 정책 | 동작 |
| --- | --- |
| `refresh_duration` | 지속시간을 초기화 |
| `keep` | 남은 지속시간 유지 |

## 주기 효과 (periodicEffects)

`interval` 마다 효과가 반복 실행됩니다. 화상 같은 도트 데미지에 사용합니다.

```json
"periodicEffects": [
  { "type": "damage", "power": 0.3, "interval": 1.0, "damageType": "fire" }
]
```

타이머는 누적 방식이라 프레임 드랍이 있어도 `interval` 기준 발동 횟수가 보존됩니다. 진행은 `system.Tick(deltaTime)` 이 담당하므로 매 프레임 호출해야 합니다.

## 스탯 수정자 (statModifiers)

버프가 살아있는 동안 유닛 스탯을 변경합니다.

```json
"statModifiers": [
  { "statKey": "attack", "modifierType": "percent_add", "value": 0.2 }
]
```

`stack` 정책 버프는 스택 수만큼 수정자가 배율 적용됩니다. 예를 들어 공격력 +20% 버프가 3스택이면 `percent_add` 합이 0.6 이 됩니다.

```text
최종 attack = (base + flat합) × (1 + percent_add합) × Π(1 + percent_mul)
```

각 버프 인스턴스가 자신을 source 토큰으로 등록하므로, 해제 시 해당 버프가 건 수정자만 정확히 제거됩니다.

## 화상 버프 전체 예시

```json
{
  "id": "burn",
  "name": "화상",
  "duration": 5.0,
  "maxStack": 3,
  "stackPolicy": "stack",
  "refreshPolicy": "refresh_duration",
  "statModifiers": [],
  "periodicEffects": [
    { "type": "damage", "power": 0.3, "interval": 1.0, "damageType": "fire" }
  ]
}
```

- 5초 지속, 최대 3스택
- 1초마다 `공격력 × 0.3` 화염 데미지
- 재적용하면 스택이 쌓이고 지속시간이 초기화

## 죽은 유닛 정리

`Tick` 중 소유 유닛이 사망하면 해당 유닛의 버프 인스턴스는 자동으로 정리되어 활성 목록에서 제거됩니다.
