---
title: Data Format
description: Skill and buff JSON schema and supported type reference.
---

Skills and buffs are defined in `skills.json` and `buffs.json` respectively. The root supports both an object wrapper (`{ "skills": [...] }`) and a top-level array (`[...]`).

## SkillDefinition

```json
{
  "id": "fire_slash",
  "name": "Fire Slash",
  "cooldown": 5.0,
  "castTime": 0.3,
  "targeting": { "type": "cone", "range": 5.0, "angle": 60, "maxTargets": 5 },
  "conditions": [{ "type": "alive" }],
  "effects": [
    { "type": "damage", "power": 1.8, "damageType": "physical" },
    { "type": "add_buff", "buffId": "burn", "duration": 5.0 },
    { "type": "play_vfx", "vfxId": "fire_slash_vfx" }
  ]
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique identifier (required, no duplicates) |
| `name` | string | Display name |
| `cooldown` | float | Cooldown in seconds. Must not be negative. |
| `castTime` | float | Cast time in seconds. Must not be negative. |
| `targeting` | object | Targeting definition |
| `conditions` | array | List of execution conditions (AND) |
| `effects` | array | Effect list (at least one required) |

## Targeting

```json
{ "type": "cone", "range": 5.0, "radius": 0, "angle": 60, "maxTargets": 5 }
```

| Field | Description |
| --- | --- |
| `type` | Targeting method (see table below) |
| `range` | Range for cone |
| `radius` | Radius for circle |
| `angle` | Cone arc angle in degrees |
| `maxTargets` | Maximum number of targets. Unlimited if 0 or less. |

| `type` | Behavior |
| --- | --- |
| `self` | The caster themselves |
| `single_enemy` | One input target |
| `circle` | Enemies within radius (requires registry) |
| `cone` | Enemies within range and angle (requires registry) |

## Condition

Conditions are checked before effects execute. If any condition fails, the skill does not execute.

```json
{ "type": "chance", "value": 0.3 }
{ "type": "hp_below", "value": 0.5 }
{ "type": "has_buff", "key": "burn" }
```

| `type` | Fields used | Meaning |
| --- | --- | --- |
| `always` | — | Always passes |
| `alive` | — | Passes if there is at least one alive target |
| `chance` | `value` | Probability (0–1) |
| `hp_below` | `value` | A target exists with HP ratio ≤ value |
| `hp_above` | `value` | A target exists with HP ratio ≥ value |
| `has_buff` | `key` | A target exists that has the specified buff |
| `not_has_buff` | `key` | All targets lack the specified buff |
| `target_count_above` | `value` | Target count ≥ value |
| `target_count_below` | `value` | Target count ≤ value |

## Effect

```json
{ "type": "damage", "power": 1.8, "value": 0, "damageType": "physical" }
{ "type": "heal", "power": 2.0 }
{ "type": "add_buff", "buffId": "burn", "duration": 5.0 }
{ "type": "play_vfx", "vfxId": "fire_slash_vfx" }
```

| `type` | Fields used | Meaning |
| --- | --- | --- |
| `damage` | `power`, `value`, `damageType` | Damage = `value + attack × power` |
| `heal` | `power`, `value` | Heal = `value + attack × power` |
| `add_buff` | `buffId`, `duration` | Apply buff (`duration` 0 uses buff default) |
| `remove_buff` | `buffId` | Remove buff |
| `play_vfx` | `vfxId` | Play VFX |
| `play_sfx` | `sfxId` | Play SFX |

## BuffDefinition

```json
{
  "id": "burn",
  "name": "Burn",
  "duration": 5.0,
  "maxStack": 3,
  "stackPolicy": "stack",
  "refreshPolicy": "refresh_duration",
  "statModifiers": [],
  "onApplyEffects": [],
  "periodicEffects": [
    { "type": "damage", "power": 0.3, "interval": 1.0, "damageType": "fire" }
  ],
  "onRemoveEffects": []
}
```

| Field | Description |
| --- | --- |
| `duration` | Duration in seconds. Must be greater than 0. |
| `maxStack` | Maximum stacks. 1 or more. |
| `stackPolicy` | `stack` / `ignore` / `replace` |
| `refreshPolicy` | `refresh_duration` / `keep` |
| `statModifiers` | List of stat modifiers |
| `onApplyEffects` | Executed once on application |
| `periodicEffects` | Repeated every `interval` |
| `onRemoveEffects` | Executed once on removal |

## StatModifier

```json
{ "statKey": "attack", "modifierType": "percent_add", "value": 0.2 }
```

| `modifierType` | Calculation |
| --- | --- |
| `flat` | base + value |
| `percent_add` | × (1 + Σ value) — additive among same stat |
| `percent_mul` | × (1 + value) — multiplicative |

Final value = `(base + sum of flat) × (1 + sum of percent_add) × Π(1 + percent_mul)`.

For detailed behavior of buff policies and stat stacking, see the [Buff System](/buffs) documentation.
