## Multiplication into addition

Building a dedicated system for each domain (SkillSystem, MonsterSystem, ‚Ä¶) means
filling in the `domain √ó behavior` matrix every time. AchReactive makes the **behavior pool
shared by all domains**, turning this into `domain + behavior`.

- Adding a new domain ‚Üí 1 data class (0 system code)
- Reusing an existing behavior ‚Üí 0 code, data only
- A genuinely new behavior ‚Üí 1 function, then shared by all domains

## The four layers

```
Data Layer       IData ¬∑ IDataLoader<T> ¬∑ DataBase<T>      (generic, provided by the package)
Reaction Layer   Reaction ¬∑ ReactionEngine ¬∑ Registry      (generic, provided by the package)
Behavior Pool    [Effect("damage")] [Condition("alive")]   (written by the game, domain-agnostic)
Domain Data      SkillData ¬∑ MonsterData ¬∑ ItemData         (data, code-gen capable)
```

The top two layers are provided by the package and don't change. You only fill in the bottom two layers.

## Reaction = trigger ‚Üí condition ‚Üí effect

Every domain is expressed in the same language.

```
Reaction
 ‚îú‚îÄ trigger      "on_use" | "on_hit" | "on_hp_below" ...
 ‚îú‚îÄ conditions   evaluated with AND (alive, hp_below, chance ...)
 ‚îî‚îÄ effects      executed in order (damage, heal, add_modifier ...)
```

The shape is identical even across different domains.

```json
// Skill ‚Äî damage on use
{ "trigger": "on_use",
  "effects": [{ "type": "damage", "params": { "power": 1.8 } }] }

// Monster ‚Äî enrage when HP is at or below 30%
{ "trigger": "on_hp_below", "triggerParams": { "threshold": 0.3 },
  "effects": [{ "type": "add_modifier", "params": { "stat": "atk", "modifier": "percent_add", "value": 0.5 } }] }

// Item ‚Äî aura on equip
{ "trigger": "on_equip",
  "effects": [{ "type": "add_modifier", "params": { "stat": "def", "modifier": "flat", "value": 5 } }] }
```

`add_modifier` is **one and the same code** whether a skill or an item uses it.

## Integration boundary

The game only implements two interfaces.

- **`IEntity`** ‚Äî `Id`, `Position`, `Stats` (StatSheet), `IsAlive`
- **`IWorld`** ‚Äî `Spawn`, `Collect`

Effects/conditions access `Source`/`Target`/`Targets`/`World` through the `ReactionContext`.
