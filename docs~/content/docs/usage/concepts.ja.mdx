## 掛け算を足し算へ

ドメインごとに専用システム（SkillSystem、MonsterSystem、…）を作ると、
`도메인 × 동작` マトリクスを毎回埋めなければなりません。AchReactive は**動作プールをすべてのドメインで
共有**させ、これを `도메인 + 동작` に変えます。

- 新しいドメインの追加 → データクラス1個（システムコード0）
- 既存の動作の再利用 → コード0、データのみ
- 本当に新しい動作 → 関数1個、以降はすべてのドメインが共有

## 4つのレイヤー

```
Data Layer       IData · IDataLoader<T> · DataBase<T>      (제네릭, 패키지 제공)
Reaction Layer   Reaction · ReactionEngine · Registry      (제네릭, 패키지 제공)
Behavior Pool    [Effect("damage")] [Condition("alive")]   (게임 작성, 도메인 무관)
Domain Data      SkillData · MonsterData · ItemData         (데이터, 코드젠 가능)
```

上の2つのレイヤーはパッケージが提供し、変わりません。下の2つのレイヤーだけを埋めれば済みます。

## リアクション = トリガー → 条件 → 効果

すべてのドメインは同じ言語で表現されます。

```
Reaction
 ├─ trigger      "on_use" | "on_hit" | "on_hp_below" ...
 ├─ conditions   AND 로 평가 (alive, hp_below, chance ...)
 └─ effects      순서대로 실행 (damage, heal, add_modifier ...)
```

ドメインが異なっても形は同じです。

```json
// 스킬 — 사용 시 피해
{ "trigger": "on_use",
  "effects": [{ "type": "damage", "params": { "power": 1.8 } }] }

// 몬스터 — HP 30% 이하에서 광폭화
{ "trigger": "on_hp_below", "triggerParams": { "threshold": 0.3 },
  "effects": [{ "type": "add_modifier", "params": { "stat": "atk", "modifier": "percent_add", "value": 0.5 } }] }

// 아이템 — 장착 시 오라
{ "trigger": "on_equip",
  "effects": [{ "type": "add_modifier", "params": { "stat": "def", "modifier": "flat", "value": 5 } }] }
```

`add_modifier` は、スキルが使おうがアイテムが使おうが**同じコード一つ**です。

## 統合境界

ゲームは2つのインターフェイスだけを実装します。

- **`IEntity`** — `Id`, `Position`, `Stats`（StatSheet）, `IsAlive`
- **`IWorld`** — `Spawn`, `Collect`

effect/condition は `ReactionContext` を通じて `Source`/`Target`/`Targets`/`World` にアクセスします。

## typeof を使わない

複数のドメインを束ねる際に、`Dictionary<Type, object>` のような型キーのサービスロケーターを使いません。
各 `DataBase<T>` を**明示的に保持**し、クロスドメイン参照が必要な effect は
[手動登録 + クロージャキャプチャ](/usage/extending#크로스-도메인)で解決します。

```csharp
public sealed class GameDataContext
{
    public readonly DataBase<SkillData>   Skills   = new();
    public readonly DataBase<MonsterData> Monsters = new();
}
```
